GitHub Pages 托管指南

把本地 HTML 变成公网可访问的网页

GitHub Pages 是什么

GitHub Pages 是 GitHub 提供的免费静态网页托管服务。你把 HTML、CSS、JS 文件推送到 GitHub 仓库，它自动将这些文件部署为一个可以通过公网 URL 访问的网站。

简单说就是：你只管提交代码，GitHub 帮你把网页挂到互联网上，无需买服务器、不用配 Nginx、不用备案域名。

特点说明
免费无限流量，仓库大小限制 1GB
自动部署推送代码后自动更新网站
HTTPS自带 SSL 证书，浏览器不会报不安全
自定义域名可以绑定自己的域名（可选）
局限只能托管静态文件（HTML/CSS/JS），不能跑后端

怎么启用

启用 GitHub Pages 就是告诉 GitHub：把哪个分支的哪个目录当成网站根目录。

方法一：调用 GitHub API（本次采用的方式）

curl -X POST "https://api.github.com/repos/{owner}/{repo}/pages" \
  -H "Authorization: token {你的token}" \
  -H "Accept: application/vnd.github+json" \
  -d '{"source":{"branch":"master","path":"/"}}'

这里 source.branch=master 指定用 master 分支，source.path=/ 表示把仓库根目录作为网站根目录。API 返回的 html_url 就是你的网站地址。

方法二：在 GitHub 网页上操作

进入仓库 → Settings → Pages → "Build and deployment" 下选择 Deploy from a branch → 选 master 分支和 / (root) 目录 → 点击 Save。

方法三：使用 gh CLI

gh api /repos/zhangxian371/ai-notebook/pages \
  -X POST -f "source[branch]=master" -f "source[path]=/"
本次实际用的是方法一（curl + API），因为电脑上没装 gh CLI。
认证用的 token 是从 git 凭据管理器自动获取的：printf "protocol=https\nhost=github.com\n" | git credential fill

背后的原理

启用 Pages 后，GitHub 在后端做了这几件事：

1. 触发构建 —— 每次你 git push 到 master 分支，GitHub 会运行一个轻量的构建流程（Jekyll 或直接静态部署）。

2. 生成静态站点 —— 因为本项目是纯 HTML，构建就是直接把文件原样拷贝到部署目标。

3. 分配到子域名 —— 部署完成后，内容被放到 https://{用户名}.github.io/{仓库名}/ 目录下，由 GitHub 的 CDN 分发。
