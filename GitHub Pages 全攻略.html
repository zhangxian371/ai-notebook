GitHub 网页托管（GitHub Pages）是怎么回事？

简单来说，GitHub Pages 是 GitHub 提供的一项免费静态网页托管服务。

平时你写好了前端网页（HTML、CSS、JavaScript），只能在自己的电脑上双击打开。如果你想让全世界的人都能通过网址访问你的网页，正常流程需要买服务器、买域名、配置环境……极其折腾。而 GitHub Pages 把这个流程简化到了极致：你只需要把网页代码上传到 GitHub 仓库，它就会自动帮你生成一个专属的公开网址。

它的核心特点：

  • 完全免费： 零成本，不需要服务器，也不强制要求买域名（但支持绑定自定义域名）。
  • 只支持静态内容： 它可以完美运行纯前端的 HTML、CSS、JS，或者由 Hexo、Hugo、VuePress 等生成的静态博客。因为没有后端数据库，所以不能运行 PHP、Java、Python 后端程序。
  • 自动化更新： 只要你修改了代码并推送到 GitHub，网站内容就会在几分钟内自动同步更新。

如何操作？手把手带你上线第一个网页

只需以下 5 个步骤，你的网页就能传遍全网：

  1. 第一步：创建一个公开仓库（Repository）

    登录你的 GitHub 账号，点击右上角的 “+” 号，选择 New repository。给仓库起个名字（例如：my-website）。注意： 仓库的权限必须选择 Public（公开），否则免费用户无法开启网页托管功能。

  2. 第二步：上传你的网页文件

    在新建的仓库里，创建一个名为 index.html 的文件（重要：网站的主首页文件名必须是 index.html，全小写）。在里面随便写点 HTML 代码，比如：
    <h1>我的第一个 GitHub 网站上线啦！</h1>
    然后保存并提交（Commit）代码。

  3. 第三步：找到 GitHub Pages 设置

    进入该仓库的主页面，点击顶部一排选项卡最右侧的 Settings（设置）。在左侧侧边栏中找到 Code and automation 分组，点击地下的 Pages 选项。

  4. 第四步：配置分支并保存

    在 "Build and deployment" 下方的 "Source" 保持默认的 Deploy from a branch。然后在下面的 "Branch"（分支）下拉菜单中，选择你的主分支（通常是 main 或 master），后面的目录保持 /root 不变，最后点击 Save 按钮。

  5. 第五步：见证奇迹的时刻

    点击保存后，GitHub 会在后台自动帮你部署。稍等 1 到 2 分钟，刷新这个 Pages 页面，你会在顶部看到一行绿色的提示语：
    "Your site is live at https://你的用户名.github.io/仓库名/"。
    点击这个链接，全世界任何人都能看到你刚才写的网页了！

    6. 💡 进阶彩蛋（如何去掉网址后面的仓库名）：
    如果你想让自己的专属网址看起来更专业（形如 https://username.github.io），你可以创建一个特殊的仓库，仓库名必须严格等于：你的GitHub用户名.github.io。在这个仓库里开启 Pages 服务后，它就是你的个人顶级主页了！
